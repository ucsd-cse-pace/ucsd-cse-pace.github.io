Choose one AI coding tool before starting a workshop. Hosted tools are the simplest path for most learners; the local Qwen path is available for users who need a local model and have the required hardware.

Option A

OpenAI Codex

Codex CLI runs in your terminal. The official release guidance highlights macOS and Linux binaries, so Windows workshop users should use WSL2 Ubuntu unless native Windows support has been confirmed for their environment.

Official references: OpenAI Help Center and OpenAI Codex GitHub.

Windows

Use this only if your workshop environment explicitly supports native Windows. For the standard workshop path, use WSL Ubuntu below.

  1. Install Node.js 18 or newer from the official Node.js installer.
  2. Open PowerShell in your project folder.
  3. Install and launch Codex.
npm install -g @openai/codex
codex

If native Windows install or shell behavior is inconsistent, move the project into WSL2 Ubuntu and follow the WSL steps.

Windows WSL Ubuntu

  1. Install WSL2 with Ubuntu from PowerShell.
  2. Open Ubuntu and update packages.
  3. Install Node.js 18 or newer, then install Codex.
  4. Run Codex from the root of your workshop project.
wsl --install -d Ubuntu
sudo apt update
sudo apt install -y git curl
node --version
npm --version
npm install -g @openai/codex
cd ~/workshops/your-project
codex

Mac OS

  1. Install Homebrew or Node.js 18 or newer.
  2. Install Codex with Homebrew or npm.
  3. Run Codex from the workshop project folder and sign in when prompted.
brew install --cask codex
codex

npm install -g @openai/codex

Option C

Local Qwen3.6-27B with llama.cpp and TurboQuant

This path runs a local TurboQuant GGUF of Qwen/Qwen3.6-27B. It avoids hosted model calls, but it is the most hardware-sensitive setup in the workshop. Start with TQ4_1S on a 24 GB or larger GPU, or use TQ2_0 / TQ1_0 on tighter hardware with reduced quality expectations.

References: official Qwen3.6-27B model card, TurboQuant GGUF model card, TurboQuant llama.cpp fork, and upstream llama.cpp build docs.
TurboQuant support is not the default upstream llama.cpp path. Use the TurboQuant fork linked by the model card when running TQ* files.

Build the TurboQuant llama.cpp fork

Use the fork referenced by the TurboQuant model card for TQ1_0, TQ2_0, TQ3_1S, and TQ4_1S files.

git clone https://github.com/kmbandy/llama.cpp
cd llama.cpp
cmake -B build
cmake --build build --config Release -j 8

For NVIDIA CUDA or AMD ROCm/HIP acceleration, follow the backend-specific CMake flags in the fork or upstream llama.cpp build guide before running the build command.

Download a TurboQuant GGUF

Download one GGUF file from mad-lab-ai/Qwen3.6-27B-tq-gguf. The model card lists these practical choices.

FileApprox. sizeUse when
Qwen3.6-27B-TQ4_1S.gguf17 GBBest workshop default if your GPU has enough memory
Qwen3.6-27B-TQ3_1S.gguf13.5 GBBalanced fallback for smaller GPUs
Qwen3.6-27B-TQ2_0.gguf7 GBLow-memory experimentation
Qwen3.6-27B-TQ1_0.gguf5.7 GBLast-resort memory fit, lowest quality
 
mkdir -p ~/models/qwen3.6-27b-tq
cd ~/models/qwen3.6-27b-tq
wget https://huggingface.co/mad-lab-ai/Qwen3.6-27B-tq-gguf/resolve/main/Qwen3.6-27B-TQ4_1S.gguf

Run as a local API server

Server mode gives coding tools an OpenAI-compatible local endpoint. Start with a 32K context while validating hardware; increase only after confirming memory headroom.

cd ~/llama.cpp
./build/bin/llama-server \
  --model ~/models/qwen3.6-27b-tq/Qwen3.6-27B-TQ4_1S.gguf \
  -ngl 999 \
  --ctx-size 32768 \
  --cache-type-k q8_0 \
  --cache-type-v q8_0 \
  --flash-attn \
  --no-mmap \
  --host 127.0.0.1 \
  --port 8081

If the server fails with an out-of-memory error, reduce --ctx-size, switch to TQ3_1S or TQ2_0, or lower GPU offload by reducing -ngl.

Quick CLI test

Use a short prompt to confirm the quant loads before connecting a coding assistant.

./build/bin/llama-cli \
  --model ~/models/qwen3.6-27b-tq/Qwen3.6-27B-TQ4_1S.gguf \
  -ngl 999 \
  --ctx-size 8192 \
  --cache-type-k q8_0 \
  --cache-type-v q8_0 \
  --flash-attn \
  --no-mmap \
  -p "Create a three-step plan for a task manager app."
