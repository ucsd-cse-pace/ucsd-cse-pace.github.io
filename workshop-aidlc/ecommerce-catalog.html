E-Commerce Product Catalog

Duration: 2-3 hours | Build a local-first full-stack product catalog using AI-DLC methodology

This workshop demonstrates incremental unit-by-unit development. Build, test, and validate each unit before moving forward.

Workshop Overview

Stack: React / Vite / FastAPI / SQLite / Docker Compose

Apply AI-DLC methodology to build a local-first e-commerce product catalog. Customers can browse products, search, view details, and add items to a SQLite-backed cart. Admins can manage products and inventory without login. Product images use static placeholder image URLs.

Prerequisites

  • Basic programming knowledge
  • Access to AI assistant (Kiro, Claude, Cursor, etc.)
  • Node.js LTS and npm
  • Python 3.11+, venv, and pip
  • Browser for local frontend and API docs validation
  • Docker Compose for the Operations packaging step
  • Code editor

Construction Activity 6: Code Generation (~60 minutes)

This is less explicit than a manual tracker (like the one in the Copy-Paste Prompts tab), but the workflow handles the cycling automatically.

Operations Activity 7: Local Full-Stack Packaging (~20 minutes)

The AI packages the local full-stack application for repeatable Operations use. This covers:

  • Backend Dockerfile for the FastAPI service
  • Frontend Dockerfile or Nginx static container for the React build
  • docker-compose.yml exposing the backend at http://localhost:8000 and frontend at http://localhost:3000
  • Persistent SQLite storage under ./data/catalog.db
  • .env.example, local deployment guide, and validation report
💡 Think about your product catalog: Docker Compose is for the Operations step. The first development path can run backend and frontend directly, while the packaged path proves the full stack can start consistently on another machine.
📋 The AI will present approval gates at each step — review and approve to continue, or request changes if needed.
