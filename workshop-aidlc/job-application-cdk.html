Local Job Application Website

Build a frontend-only job application site using React, Vite, seeded data, and localStorage

Follow the complete AI-DLC methodology to build a local-first job application site with open-source frontend tooling.

Workshop Overview

Duration: ~3 hours | Project: Job Application Site | Stack: React / Vite / localStorage

Build a frontend-only job application management site that allows applicants to search, view, and apply to seeded job postings. Submitted applications are stored in browser localStorage. Docker Compose packaging is reserved for the Operations step.

Prerequisites

  • Basic React/JavaScript knowledge
  • Access to an AI assistant (Kiro, Claude, Cursor, etc.)
  • Node.js LTS and npm installed
  • Browser for local preview and localStorage testing
  • Docker Compose for the Operations packaging step
  • Code editor

  • Using AI-DLC Workflow
  • Copy-Paste Prompts

Operations Activity 7: Local Static Packaging (~30 minutes)

The AI packages the frontend-only React/Vite app for repeatable local Operations use. This covers:

  • A production Vite build under job-app/dist
  • A Dockerfile that serves the static build with Nginx
  • A docker-compose.yml exposing the site at http://localhost:3000
  • An optional nginx.conf for SPA routing fallback
  • .env.example, local deployment guide, and validation report
💡 Think about your job application site: Docker Compose is for the Operations step. The development path can use Vite preview, while the packaged path proves the app can run as a static site on another machine.
